# Documentation

Эта папка содержит рабочую документацию по курсовому проекту АИС "Клиника".

## Цель

Фиксировать процесс разработки так, чтобы в любой момент было видно:
- что именно сделано;
- зачем это сделано;
- какой результат получен;
- что делать следующим шагом.

## Состав

- `PROJECT_STRUCTURE.md` — целевая файловая структура проекта (backend/frontend/infra/docs).
- `WORKLOG.md` — пошаговый журнал работ в формате "что сделано / зачем / результат".
- `ER_MODEL.md` — описание ER-модели и правил целостности.
- `API_ENDPOINTS_AND_CONTRACTS.md` — взаимодействия по C4 L3, перечень эндпоинтов и JSON-контракты.
- `openapi.yaml` — машиночитаемая спецификация OpenAPI 3.0.3 для Swagger.

## Правила ведения

1. Каждое значимое изменение фиксируется отдельной записью в `WORKLOG.md`.
2. У записи обязательно должны быть:
   - дата;
   - действие;
   - обоснование (зачем);
   - артефакты (какие файлы созданы/изменены).
3. Если меняется архитектура или состав модулей, обновляется `PROJECT_STRUCTURE.md`.
4. Если меняются API-контракты или RBAC-правила, синхронно обновляются:
   - `Documentation/API_ENDPOINTS_AND_CONTRACTS.md`;
   - `Documentation/openapi.yaml`;
   - разделы запуска/проверки в корневом `README.md`.
5. Документация обновляется в тот же рабочий шаг, в котором внесены изменения в код (без отложенных правок "потом").
6. Предпочтительный уровень детализации:
   - не только "что изменилось", но и "как проверить" и "какие ограничения/ошибки ожидаемы";
   - в каждом модуле отдельно указывать статусы реализации: `implemented`, `partial`, `planned`.
